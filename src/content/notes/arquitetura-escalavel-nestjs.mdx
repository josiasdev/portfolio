# Construindo Sistemas Escaláveis com NestJS e Docker

A construção de plataformas modernas exige não apenas conhecimento em frameworks, mas também uma compreensão sólida de infraestrutura e padrões de projeto. Neste artigo, vamos explorar a arquitetura por trás de uma plataforma **Buy Now, Pay Later** (BNPL) na blockchain Stellar.

## Por que NestJS?

NestJS fornece uma arquitetura baseada no Angular, promovendo o uso massivo de **Injeção de Dependência** e tipagem estática rigorosa através do TypeScript. 

```typescript
import { Controller, Get } from '@nestjs/common';
import { AppService } from './app.service';

@Controller('transactions')
export class TransactionsController {
  constructor(private readonly appService: AppService) {}

  @Get()
  getTransactions() {
    return this.appService.getRecentTransactions();
  }
}
```

### Principais Vantagens:
1. **Testabilidade:** A injeção de dependência facilita o mock de serviços durante testes unitários.
2. **Escalabilidade:** Módulos altamente desacoplados.
3. **Ecossistema:** Integração transparente com TypeORM, Prisma, Mongoose e GraphQL.

## Integração com a Blockchain Stellar

Diferente do Ethereum que utiliza a EVM, a Stellar é focada em velocidade e baixo custo para pagamentos. Na SyloPay, o desafio era criar **Smart Contracts** leves (usando Soroban) que garantissem a liberação de fundos para o comerciante instantaneamente.

> "A Stellar não foi projetada para ser um computador mundial, mas sim a melhor rede global de pagamentos."

## Docker: Consistência entre ambientes

Não importa o quão bom seja seu código se ele rodar apenas na sua máquina. Empacotar a aplicação num contêiner `Dockerfile` multi-stage nos garantiu que a imagem final teria menos de 150MB, melhorando o cold-start nos servidores de produção.

Espero que essa nota técnica ajude você a considerar novas arquiteturas nos seus próximos projetos Backend!
